Rappel au sujet de la gestion de version

Durée : 1 h 30

Prérequis : notions en Flask

Contexte :

Flask est un Framework Web qui facilite la création des applications Web complètes à l'aide de Python. Mais une fois que nous créons nos applications, nous voulons les mettre à la disposition de l'utilisateur final. Ce cours présente comment nous pourrions déployer nos applications développées avec Flask à l'aide de Heroku et Github. Dans un premier temps, nous verrons ce qu'est la gestion des versions avec Git et Github, puis nous présenterons brièvement Heroku et finalement nous verrons comment nous pourrions réussir à déployer nos applications Flask à l'aide de ces outils.

Objectifs

  • Connaître la notion de gestion de version

  • Comprendre Git et Github

  • Connaître les principales commandes de Git

Contexte :

Dans le contexte d'une pratique professionnelle, il vous est demandé d'avoir la notion de version de contrôle, et de travailler avec des outils dédiés pour sa gestion comme Git, afin de pouvoir collaborer avec d'autres développeurs.

En tant que développeur, connaître et savoir manipuler Git et Github sont indispensables. Dans cette partie nous allons comprendre comment gérer nos applications en utilisant Git et Github.

Git

Git est un système de contrôle de version open source et gratuit qui est largement utilisé auprès de la communauté des développeurs et dans les entreprises. Git est un outil en ligne de commande. La question qui se pose est « qu'est-ce que le contrôle de version ? ».

Contrôle de version

Les développeurs et les data scientists ont tendance à apporter des modifications au code lors du processus de création de leurs logiciels et applications. Ceci conduit à la publication de plusieurs versions, même après une version officielle.

Le contrôle de version aide à gérer les modifications apportées aux documents, aux logiciels, aux sites web et autres créations informatiques. Ainsi, chaque modification de code est stockée dans un dépôt central, que nous pouvons consulter pour voir toutes les modifications en fonction du temps. Ceci nous permet de facilement faire le suivi des bogues et revenir à une version précédente du code lorsque l'on en a besoin.

Installation de Git

La première chose à faire pour pouvoir utiliser Git est de l'installer. Voici un petit guide d'installation pour les systèmes d'exploitation les plus populaires.

  • Installation sous Linux :

    Pour une distribution Fedora ou toute distribution basée sur RPM comme RHEL ou CentOS, nous utilisons dnf. Dans le terminal on exécute la commande :

    sudo dnf install git-all

    Pour une distribution basée sur Debian comme Ubuntu, nous utilisons apt. Dans le terminal on exécute la commande :

  • sudo apt install git-all

  • Installation sous MacOS :

    Pour installer une version à jour de Git sur un Mac, nous passons par le programme d'installation à télécharger sur git.

  • Installation sous Windows :

    Sous Windows, nous passons également par le programme d'installation disponible sur git.

Commandes Git pour des dépôts locaux

Comme Git est un outil en ligne de commandes, il faut connaître les commandes les plus fréquentes pour pouvoir l'utiliser. Nous allons dans cette section voir les principales commandes Git pour travailler avec des dépôts locaux :

Par dépôt local, nous entendons le dossier de l'ordinateur que nous souhaitons déplacer vers Github.

La première commande à utiliser est : git init

Cette commande permet de transformer un répertoire local en un dépôt (repo) Git vide. C'est la première étape de la création d'un dépôt. Après l'exécution de cette commande, il nous est possible d'ajouter des fichiers et des répertoires à notre dépôt. En effet, nous initialisons un dépôt Git à la racine du répertoire. Si on vérifie notre dossier nous allons trouver un sous-dossier.git.

git add : c'est la commande utilisée pour ajouter des fichiers à la zone de transit pour Git. Avant qu'un fichier ne soit disponible pour être validé dans un dépôt, le fichier doit être ajouté à l'index Git (zone de transit).

Nous pouvons utiliser cette commande sous plusieurs formes :

  • git add : pour ajouter tous les fichiers ne figurant pas dans la zone de transit.

  • git add mon_fichier.txt : pour ajouter un fichier en particulier.

  • git add mon_dossier : pour ajouter un dossier entièrement.

  • git status : cette commande aide à connaître l'état de nos fichiers, en zone de transit, validés, etc.

  • git commit : c'est la commande qui permet d'enregistrer les modifications apportées aux fichiers dans un dépôt local. Il est recommandé d'inclure un message décrivant les modifications apportées à chaque commit : git commit -m "Message du commit"

La prochaine chose à faire serait de transférer les modifications apportées à un dépôt distant (sur Github).

Github

GitHub est une plate-forme d'hébergement de code pour le contrôle de version et nous permet de travailler sur des projets avec des gens de part et d'autre dans le monde. En d'autres termes, c'est là où les développeurs peuvent stocker leurs projets et créer un réseau avec des personnes partageant les mêmes idées. Nous allons d'abord visiter Why GitHub?.

Création d'un dépôt

Une fois que nous avons un compte Github, nous pouvons facilement créer un dépôt. Quand nous sommes connectés, nous pouvons visiter GitHub pour créer un nouveau dépôt.

Une fois cela fait, nous devrons remplir le formulaire qui apparaît, nous remplirons le nom du dépôt, une petite description (facultative), choisir si nous voulons un dépôt privé ou public, nous pouvons également ajouter un fichier README, etc. Quand nous terminons de remplir le formulaire, nous cliquons sur le bouton « Create repository ».

Commandes Git pour des dépôts distants

Une fois le dépôt créé, il existe en tant que dépôt distant. Nous pouvons cloner notre dépôt pour créer une copie locale sur notre ordinateur et synchroniser entre les deux emplacements.

Pour créer une copie local d'un dépôt distant existant, nous utilisons git clone qui nous permettra de copier le dépôt sur notre ordinateur. Git crée un répertoire local contenant tous les fichiers.

git clone <URL_du_depot_distant>

Une fois le dépôt cloné nous pouvons y accéder sur notre ordinateur et apporter les modifications que nous souhaitons, nous allons utiliser les commandes git add, git commit vues précédemment. Une fois cela fait, la prochaine étape est de mettre ces modifications dans notre dépôt distant sur GitHub.

git push est la commande utilisée pour envoyer des commits locaux au dépôt distant. Il nécessite deux paramètres : le dépôt distant et la branche à laquelle le push est destiné.

git push <URL_distant> <branche>
